Both projects split analog and digital control timing. EDGE goes further because the external encoder and button surface need deterministic debounce independent of display and UI work.

Architecture

The architecture view is split by comparison type: Pod-vs-Pod for Pod control-surface decisions, Field-vs-Field for Field control/state decisions, and cross-family only for timing ownership, DSP/runtime shape, state handoff, and readiness.

POD_EDGE_mono_DSP Structure

  • Board target: Daisy Pod plus external I2C OLED, external encoder, BAK, and CON controls.
  • Control split: digital controls in a 1 kHz TIM_5 callback; analog controls in `AudioCallback`.
  • DSP: input conditioning, soft drive, SDRAM delay, feedback filtering, diffusion, tone tilt, limiter.
  • UI: six OLED pages rendered only when state changes and capped by elapsed time.
  • Thread safety: main loop writes inactive `FxParams`, then swaps `active_buf_` after `__DSB()`.

Pod_MultiFX_Chain Structure

  • Board target: plain Daisy Pod control surface with no external OLED board.
  • Control split: analog controls in `AudioCallback`; encoder/buttons in the main loop.
  • DSP: `Input -> LFO -> Tube -> Delay -> Reverb -> Output`.
  • UI: four effect pages selected by encoder and indicated by RGB LEDs.
  • State: page, bypass, waveform, shift layer, and reattach behavior live in `pod_multifx_pages.h`.

Field_AdditiveSynth Structure

  • Board target: Daisy Field with TRS MIDI, 8 knobs, 16 key switches, two switches, OLED, and LEDs.
  • Control ownership: MIDI, Field controls, pickup/catch, keys, display, and LEDs run in the 1 ms main loop.
  • DSP: 8 additive voices, each using `HarmonicOscillator<16>` plus ADSR; global LFO, Chorus, and `ReverbSc`.
  • State model: two `PageState` pages, stored knob values, previous physical reads, and locked/captured flags.
  • Readiness signal: previously used as the known-good Field fallback and successfully flashed with STLINK.

Field_Template_April Structure

  • Board target: Daisy Field starter synth using shared `foundation_examples` Field defaults and UI helpers.
  • Control ownership: external MIDI, `hw.ProcessAllControls()`, `ProcessUi()`, LEDs, and OLED run in the 1 ms main loop.
  • DSP: one mono voice with oscillator, sub oscillator, LFO, ADSR, SVF, white noise, glide, velocity, key tracking, and output trim.
  • State model: `ParamBankSet`, active main/alt bank, hidden output level, tri-state key LED state, and focus/zoom state.
  • Reference value: cleaner Field starter shell for new display-based, banked-control projects.

Cross-Family Timing Interpretation

Pod and Field projects all use 48-sample blocks at 48 kHz, so the audio callback cadence is 1 ms. The key architecture difference is not the board family; it is where controls are owned. EDGE has an extra 1 kHz hardware timer for digital controls. The Field projects keep Field controls, MIDI, OLED, and LEDs in the 1 ms main loop while the audio callback stays focused on DSP.

Responsiveness

This page separates implemented behavior from proposed improvements. The implemented project cards describe code already present in the projects. The MultiFX improvement card lists next-step ideas, not current behavior.

Implemented in EDGE

  1. Move digital debounce into `ControlTimerCallback` at 1 kHz.
  2. Keep `pod.ProcessAnalogControls()` in `AudioCallback` at block cadence.
  3. Keep I2C, GPIO reads, print, allocation, and display work out of the audio callback.
  4. Use double-buffered `FxParams` so audio reads a coherent snapshot.
  5. Throttle OLED redraw by dirty state and elapsed time.

Implemented in MultiFX

  1. Follow BUG-005: analog controls are processed in the audio callback.
  2. Keep encoder/buttons in the main loop through `ProcessDigitalControls()`.
  3. Preserve page/layer stored values after page changes.
  4. Reattach only after movement of at least `0.02` normalized units.
  5. Slew a full-scale move over `300 ms` to avoid abrupt parameter jumps.

Potential MultiFX Improvements

  1. Add a 1 kHz timer path for digital controls if an external encoder, OLED, or heavier UI loop is added.
  2. Add a double-buffered parameter snapshot if the page state grows or DSP reads become more complex.
  3. Add measured loop-jitter logging around `ProcessDigitalControls()` and `System::Delay(1)`.
  4. Expand host tests for SW1 hold/release boundaries, bypass while shifted, and rapid page turns.
  5. Add event-rate counters for encoder increments to prove no turns are swallowed under stress.

Implemented in Field_AdditiveSynth

  1. Page switch locks all knobs on the target page before stored values can change.
  2. Pickup/catch releases a knob only after the physical value crosses the stored value or is within `0.02`.
  3. Loading a spectral preset locks Page A `K1-K3` and syncs stored values to the preset.
  4. Locked knob LEDs dim to `0.05`, making capture state visible.
  5. MIDI, controls, OLED, and LEDs run in the 1 ms main loop, not in the audio callback.

Implemented in Field_Template_April

  1. `ParamBankSet` preserves main and alt bank values independently.
  2. Bank changes require pickup/catch before a physical knob can overwrite the stored value.
  3. Knob LEDs show stored logical values and dim when uncaptured.
  4. Hidden `SW2 + K8` output level has its own capture state.
  5. Keybed state uses tri-state LEDs, keeping state feedback visible without stealing note-entry keys.

Parameters

The parameter models differ more than the DSP labels suggest: EDGE uses one broad parameter struct for a six-page instrument; MultiFX stores values per effect page and per logical layer.

EDGE Pages

  • P1 Perform: time, BPM, feedback, wet/dry, drive, freeze, delay mode.
  • P2 Tone: input HP, feedback LP/HP, wet tone/damping, output tilt.
  • P3 Motion: wow depth, wow rate, modulation enable.
  • P4 Freeze: freeze mode, hold behavior, loop size.
  • P5 Presets: load/save/init slots through external navigation.
  • P6 System: OLED brightness, encoder direction, bypass, sample rate, block size, firmware version.

MultiFX Pages

  • Tube: Drive and Mix; shifted layer Bias and Distortion.
  • Delay: Time from 10 ms to 100 ms and Feedback from 0.0 to 0.5.
  • Reverb: Decay and Mix; tone fixed at 12 kHz low-pass.
  • LFO: Amplitude coarse and Rate coarse; shifted layer Amplitude fine and Rate fine.
  • Encoder push on LFO cycles Sine, Triangle, Square.
  • Each page stores values independently and rearms reattach when selected.

Field_AdditiveSynth Parameter Model

  • Two pages: Page A for spectral/ADSR/reverb, Page B for LFO/chorus/master.
  • `PageState` stores applied values, previous physical knob values, and lock state for each knob.
  • Spectral presets update rolloff/even/odd and lock K1-K3 until physical capture.
  • Toggle state covers LFO sync, reverb bypass, chorus bypass, and sustain.
  • OLED zoom is driven by `zoom_param` and `zoom_start` after knob movement.

Field_Template_April Parameter Model

  • Main bank: cutoff, resonance, ADSR, drive, and color.
  • Alt bank: envelope amount, LFO rate/depth, glide, velocity amount, key tracking, noise, and sub.
  • Hidden level trim uses `SW2 + K8` and separate output capture state.
  • State controls cover waveform, velocity mode, key tracking, LFO target, glide mode, transpose, panic, and resets.
  • Focus state stores label/value text for the edit zoom page.
